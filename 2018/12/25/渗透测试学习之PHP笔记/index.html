cookie与session



发送给服务端，用来做验证的
用户访问服务器的时候服务器会返回给用户一个cookie记录在本地
请求网站 如果cookie没有phpsession那么服务器就会给你定义一个PHPSESSID
如果请求网站有PHPSESSID，服务器看到了PHPSESSION之后获取PHPSESSID的内容去找session的内容名字的文件名。
<?php
echo “Hello World”;
session_start(); //PHP从session仓库中加载已经存储的session变量
$_SESSION[‘name’]=’caocao’;
//session文件中的值就是name|s:6:”caocao”;
//$_SESSION 好比一个数组 $_SESSION[‘name’] = ‘caocao’ 这好比在数组中加了一个元素，相当于$_SESSION = array(“name”=>”caocao”) 使用的时候 还要使用$_SESSION[‘name’] 才能得到’caocao’。
?>

cookie跨域

默认的cookie只能对当前域名（完整域名：有效的二级域名）有效

php设置cookie：

    setcookie("PHPSISSID",session_id(),time()+100,'/',"xss.cn");

文件上传

upload_tmp_dir = D:\phpstudy\PHPTutorial\find_tmp    上传临时文件目录

enctype 属性规定在发送到服务器之前应该如何对表单数据进行编码
语法为<form enctype="value">
multipart/form-data 不对字符编码。 在使用包含文件上传控件的表单时，必须使用该值


move_uploaded_file($f, "./.jpg") 将上传的文件移动到指定位置
$_FILES["file2"]["tmp_name"]临时目录

$_FILES["file2"]["error"]他会返回0-6
0：没有错误发生文件上传成功
1：上传的文件大小超过php.ini中最大值2M(upload_max_filesize)
2:上传的文件大小超过html中MAX_FILE_SIZE选项指定的值
3：文件只有部分被上传，上传文件被中断
4：没有选中上传文件，然后点击上传
6：找不到临时文件，指定的临时目录不存在
upload_err_cant_write将文件读到目录的时候，没有权限写入（linux需要文件写入权限）

实例

一个提交框1.html

<html>
<head>
<meta charset="utf-8">
<title>文件上传</title>
</head>
<body>

<form action="./1.php" method="post" enctype="multipart/form-data">
<p>
上传：<input type="file" name="file2">
</p>
<p>
<input type="submit" name="提交" value="上传">
</p>
</form>
</body>
</html>

对上传文件的操作1.php

<?php
var_dump($_FILES);
include_once './1.html';
switch (@$_FILES["file2"]["error"]){
    case 0:
    @move_uploaded_file($_FILES["file2"]["tmp_name"], "./111.jpg"); //如果文件上传成功就把临时文件移动到当前目录然后修改名字为111.jpg
    break;
    default:
        echo 1;
        break;
}
?>
